> All sites are automatically crawled by GitHub Actions on a schedule, with data updated every 15 minutes to 8 hours.
> Site icons are automatically fetched from each website on first run and cached locally — no external icon services needed.

## Update Frequency Guide

| Frequency | Use Case | Example Sites |
|-----------|----------|--------------|
| Every 15~20 min | High frequency, flash sales & time-sensitive deals | 线报酷, 线报ICU, 专业线报, etc. |
| Every 30 min | Medium-high frequency | H6线报, 天天赚, 网赚, etc. |
| Every 1~2 hours | Regular updates | 白菜哦, 拔草哦, 慢慢买, etc. |
| Every 4~8 hours | Low frequency, slow-changing content | 反斗限免, 423Down, 果核剥壳, etc. |

::: info Night Mode
To save GitHub Actions quota, all sites automatically reduce crawl frequency during **23:00 ~ 07:00 (Beijing Time)** (interval ×3). See [Update Scheduling](/en/config/schedule) for details.
:::

::: info Icon Note
Site icons are automatically fetched from each website during the first GitHub Actions run and cached in the `public/icons/` directory. No external icon services (like Google) are needed. If a site's icon is temporarily unavailable, the system falls back to DuckDuckGo, Icon.horse, or generates an SVG placeholder with the site's initial letter.
:::

## How to Subscribe

Click the "Subscribe" link in the table above, copy the URL, and add it to your RSS reader. Not sure which reader to use? Check out [RSS Readers](/en/feeds/readers).
