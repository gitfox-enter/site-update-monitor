# 过滤与黑名单

RSSForge 提供多层过滤机制，帮你只留下真正想要的内容。

## 内容过滤规则

### 关键词过滤

在 `blacklist.json` 中添加黑名单关键词，包含该关键词的内容会被自动过滤：

```json
{
  "keywords": [
    "广告",
    "推广",
    "[招聘]",
    "【限时】"
  ],
  "domains": [],
  "patterns": []
}
```

### 正则过滤

使用正则表达式匹配更灵活：

```json
{
  "patterns": [
    "^\\[广告\\]",
    "请点击查看全文",
    "^抢.*红包"
  ]
}
```

### 域名黑名单

如果某个站点的特定子域名内容质量差，可以屏蔽：

```json
{
  "domains": [
    "ads.example.com",
    "click.example.com"
  ]
}
```

## 站点级别黑名单

在 `blacklist.json` 的 `sites` 中可以按站点禁用：

```json
{
  "sites": {
    "https://spam.example.com/": "已停用，转向新站"
  }
}
```

被屏蔽的站点会在 `crawl_status.json` 中标记为 `dead`。

## 通用解析器 vs 专用解析器

| 解析器 | 说明 | 适用场景 |
|--------|------|---------|
| **专用解析器** | 针对特定网站的 HTML 结构定制，准确率高 | `deal_sites.py` 中已收录的 40+ 站点 |
| **通用解析器** | 尝试提取所有 `<a>` 链接和标题，灵活但可能有噪音 | 不在专用列表中的网站 |

如果专用解析器提取的内容质量差，可以改用通用解析器（删除 `deal_sites.py` 中对应条目）。

## 内容去重

RSSForge 有两层去重：

### 1. URL 去重
同一 URL 的条目只保留一条，用于分页抓取时的跨页去重。

### 2. 内容相似度去重
基于标题和内容的 MD5 哈希去重，相似度超过 90% 的条目会被过滤。

```python
# 相似度计算核心逻辑（来自 common.py）
similarity = jaccard(words1, words2)
if similarity > 0.9:
    # 过滤重复内容
    pass
```

## 垃圾内容过滤

`is_junk()` 函数自动过滤以下内容：

- 纯数字或纯符号的标题
- 长度小于 3 字符的标题
- 长度超过 999 字符的标题
- 包含「点击查看」「登录」「注册」等常见垃圾词

## 最小条目数保护

部分解析器有最小条目数保护（如 `parse_423down_items` 要求至少 3 条），避免只抓到导航链接而误判为空页面。
